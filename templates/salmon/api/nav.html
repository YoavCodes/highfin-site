Navigation and Routing

Pages are defined in the configuration object passed into fin.init(). The navigate property essentially lists pages, which are defined by listing the templates to load into each container, the templates and namespaced functions to preload, and which functions to automatically execute on navigating to the page before and after the templates are rendered.

Routes are also defined in the configuration object. The routes property is a mapping of hashbang url fragments, in the format /#!/, to pages. When visiting a hashbang url, fin will recursively check for matching routes toward a blank url if none are found it will default to the default_page also defined in your initial configuration.

nav( key ) Returns: undefined
Navigates to the page specified by key.
key : String
The page defined in the configuration object.

After you navigate to a page you'll be able to access the key for the last page visited using the last_nav() method.

note: While internally it's used as a setter, you'll most likely only ever use it as a getter.

fin._meta.last_nav( key ) Returns: String
Sets or Gets the last page navigated to.
key : String
The key for the page.

Under the hood last_nav() actually keeps track of every page navigated to. While users can use your browser's back button for pages associated with a hashbang route, for pages that aren't accociated with a route you can use the go_back() function.

fin._meta.go_back( ) Returns: undefined
Navigates to the previous page.
