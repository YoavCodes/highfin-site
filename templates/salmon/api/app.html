App

The /app folder contains your back-end code. Using node's common.js module.exports pattern any .js file in your app folder and subfolders will automatically be included when the salmon node.js process is run, and they'll be namespaced under the tail.app relative to your /app folder's directory structure.

[[  print( fin.fn.escapeHtml(`

// in the file /app/auth.js

function doSomething() {
	console.log('do something');
}

module.exports = {
	doSomething: doSomething
};

`) )
	 ]]

[[  print( fin.fn.escapeHtml(`

// once your app is running, you can access the doSomething function from any server-side code

tail.app.auth.doSomething(); // prints 'do something'

`) )
	 ]]

This gives you the flexibility to structure your code in any way you like. Because the framework's namespacing mirrors your directory structure you know exactly where a function is defined and how to access it, and global availability allows you to rapidly prototype and refactor with minimal friction.
