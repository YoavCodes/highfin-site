Guppy CLI

Guppy compiles to a single binary. Put it somewhere on your PATH.

Run guppy run from any folder and it'll recurse up to parent directories until it finds a -.jsonfile. If you have multiple apps and/or execs the guppy run command will launch all of them. You can optionally specify an app by name guppy run app1

guppy run will monitor files and folders specified in your -.json for changes. Designed to work with Go and Node.js apps it can run go install for you, compile LESS, run jasmine tests for back-end or front-end using PhantomJS

Guppy can manage and generate ssh keys for your project. Run npm install and run node apps for multiple versions of node.js existing on the same server. And has shortcuts to git commit and git push your code which ties into Fishtank and Aquatic

While Guppy can run on your typical Debian or Ubuntu server just fine, it's best to run it in a fry-box. A fry-box is just a vagrant basebox where the GOPATH and other variables have been configured already and guppy is ready to use.
